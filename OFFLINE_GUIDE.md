# Workout Tracker - オフライン使用ガイド

## 📱 スマホでオフライン実行する方法

### 必要なファイル
- `index.html`
- `workout_tracker_main.js`

この2つのファイルを同じフォルダに配置してください。

---

## 🔧 セットアップ手順

### iPhone/iPad の場合

1. **ファイルを転送**
   - AirDrop、iCloud Drive、またはメール添付で2つのファイルを転送
   - ファイルアプリで同じフォルダに保存

2. **HTMLファイルを開く**
   - ファイルアプリで `index.html` をタップ
   - Safariで開く

3. **ホーム画面に追加（推奨）**
   - Safari で開いた状態で「共有」ボタンをタップ
   - 「ホーム画面に追加」を選択
   - アイコンをタップすればアプリのように起動できます

### Android の場合

1. **ファイルを転送**
   - Google Drive、またはUSBケーブルで2つのファイルを転送
   - 同じフォルダに保存

2. **HTMLファイルを開く**
   - ファイルマネージャーで `index.html` をタップ
   - Chrome で開く

3. **ホーム画面に追加（推奨）**
   - Chrome のメニュー（︙）をタップ
   - 「ホーム画面に追加」を選択
   - アイコンをタップすればアプリのように起動できます

## 🛡️ データ保護の重要性

### ⚠️ 注意事項

**データは端末のブラウザに保存されます。**以下の場合、データが失われます：

- ブラウザのキャッシュ・履歴をクリア
- アプリのデータを削除
- OSの再インストール
- 端末の初期化

### 📦 必ずバックアップを取ってください！

#### バックアップ方法

1. アプリの「Backup」ボタン（ダウンロードアイコン）をタップ
2. 「Download Backup」をタップ
3. JSONファイルがダウンロードされます
4. このファイルを安全な場所に保存：
   - iCloud Drive
   - Google Drive
   - Dropbox
   - パソコン
   - など

#### 推奨バックアップ頻度

- **週1回**: 定期的にトレーニングしている場合
- **月1回**: たまにトレーニングする場合
- **データ入力後すぐ**: 重要な記録を入力した直後

💡 **自動リマインダー**: 7日間バックアップしていない場合、アプリが通知します

#### 復元方法

1. アプリの「Backup」ボタンをタップ
2. 「Import Data」でバックアップファイルを選択
3. データが復元されます

---

## 💾 データの保存場所

- **保存先**: ブラウザの localStorage
- **容量制限**: 約5-10MB（数年分のデータを保存可能）
- **アクセス**: オフラインでも使用可能

---

## 🔄 複数端末での使用

### 同じデータを別の端末で使う場合

1. 端末Aでバックアップファイルをダウンロード
2. バックアップファイルを端末Bに転送
3. 端末Bでインポート

⚠️ **注意**: 各端末のデータは独立しています。自動同期はされません。

---

## 🆘 トラブルシューティング

### データが消えた場合

1. 最新のバックアップファイルを探す
2. アプリで「Import Data」から復元

### アプリが動かない場合

- ブラウザを最新版に更新
- キャッシュをクリア（データは残ります）
- 端末を再起動

### ファイルが開けない場合

- ファイルアプリで `index.html` を長押し
- 「共有」→「Safari/Chrome」で開く

---

## 📊 データ形式

バックアップファイルはJSON形式で、以下の情報を含みます：

```json
{
  "workouts": [...],        // ワークアウト記録
  "nutrition": [...],       // 栄養記録
  "customExercises": [...], // カスタム種目
  "exportDate": "...",      // エクスポート日時
  "version": "1.0"          // バージョン
}
```

## 💡 ベストプラクティス

1. **週1回バックアップを取る**習慣をつける
2. **複数の場所にバックアップを保存**（クラウド + ローカル）
3. **重要な記録後はすぐにバックアップ**
4. **バックアップファイル名に日付を入れる**
   - 例: `workout-tracker-backup-2024-02-15.json`
